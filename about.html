О программе MangApp

Программа предназначена для управления заявками клиентов. На данной странице Вы найдете подробное описание работы, технические подробности, а также инструкцию по эксплуатации.

Работа программы строится на базе данных (orders_log) состоящей из 4-ех таблиц:

  • cashiers - содержит данные о кассирах (ФИО, логин и пароль для аутентификации)
  • workers - содержит данные о монтажниках (ФИО, логин и пароль для аутентификации, адрес электронной почты для отправки оповещений)
  • app - содержит информацию о заявках клиентов (связывает все таблицы между собой с помощью индексов)
  • clients - содержит информацию о клиентах (ФИО, адрес, номер телефона)

На главной странице пользователю необходимо ввести имя пользователя (login) и пароль для входа в систему. Программа поочередно проверяет данные на совпадение сначала в таблице cashiers за тем в таблице workers. При совпадении задаются параметры привилегий для пользователя.

После прохождения аутентификации перед пользователем предстает главное окно программы, в зависимости от привилегий пользователя отображается основная таблица с данными заявок и основная форма ввода данных. Большинство данных программы отправляются на сервер асинхронным методом (без перезагрузки страницы). Кнопка "Записать" проверяет заполнение всех полей формы и после отправляет данные формы на сервер где они обрабатываются. При вводе данных клиента программа предлагает варианты авто заполнения. При заполнении форм ВНИМАТЕЛЬНО проверяйте вводимые Фамилию Имя и Отчество. Эти данные не подлежат редактированию и изменить их сможет лишь администратор MangApp!

ОБРАТИТЕ ВНИМАНИЕ! Если Вам необходимо создать нового клиента данные которого (ФИО) совпадает с уже зарегистрированным в системе клиентом, не нажимайте на строку с данными для авто заполнения! При нажатии программа автоматически присвоит заявку уже зарегистрированному клиенту! Если же клиент уже зарегистрирован и повторно обращается, следует нажать на строку авто заполнения для присвоения заявки к клиенту.

Кнопка "Обновить" обновляет страницу сбрасывая все заполненные данные форм!

Кнопка "Только невыполненные" отображает в основной таблице только заявки с статусом "Не выполнена". Повторное нажатие (кнопка сменит свое состояние на "Показать все") вернет в основную таблицу все заявки.

Кнопка "Выйти из системы" завершает сеанс работы и совершает выход из программы переводя пользователя на окно аутентификации.

При нажатии на строку основной таблицы пользователю открывается окно полной информации по данной заявке, а также меню управления в нижней части области полной информации.

При нажатии на кнопку "Редактирование" пользователю открывается доступ к полям редактирования основной информации по данной заявке, а также кнопка подтверждения изменений ("Изменить").

При нажатии на кнопку "Перейти к абоненту" пользователю открывается окно с подробными данными зарегистрированного абонента, а также меню редактирования основной информации абонента.

При нажатии на кнопку "Редактирование" пользователю открывается доступ к полям редактирования основной информации абонента. При нажатии кнопки "Применить" данные отправляются на сервер для обработки, а страница перезагружается.

При нажатии на кнопку "Изменить статус" в окне полной информации по заявке пользователю открывается доступ к полям редактирования статуса. При нажатии по кнопке "Применить" данные отправляются на сервер для обработки, а страница перезагружается.

Примечание! Кнопка "Изменить статус" отображается в зависимости от текущего статуса заявки! Для изменения статуса уже выполненной или отмененной заявки обратитесь к администратору MangApp!

В версии MangApp v.0.5 была добавлена функция оповещения всех монтажников о поступлении новой заявки, а также функционал, позволяющий лишь одному из монтажников принять заявку к выполнению.

Программа MangApp является разработкой tvnet3.ru
